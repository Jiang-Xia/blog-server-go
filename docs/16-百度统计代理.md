## 交付摘要

迁移 Nest `GET /resources/baidutongji` OAuth 代理与 token 缓存；Plan 12 超管接口 `POST /scheduled-task/cache/refresh-tongji-token` 委托同一 `ResourcesService`，避免双实现。

## 配置与环境

`configs/blog.yaml` / `monolith.yaml` 的 `app` 段（Nest env 对照）：

| 键 | Nest env |
|----|----------|
| `tongji_refresh_token` | `app_tongjiRefreshToken` |
| `tongji_client_id` | `app_tongjiClientId` |
| `tongji_client_secret` | `app_tongjiClientSecret` |

环境变量覆盖（Viper）：`APP_TONGJI_REFRESH_TOKEN`、`APP_TONGJI_CLIENT_ID`、`APP_TONGJI_CLIENT_SECRET`。

生产 PM2 部署：`deploy/pm2/env.production` 中 `app_tongjiRefreshToken` 等（与 Nest 同文件）经 `scripts/sync_pm2_config_from_nest.go` 写入 `deploy/pm2/configs/blog.yaml` 的 `app.tongji_*`；改 env 后执行 `go run ./scripts/sync_pm2_config_from_nest.go --env deploy/pm2/env.production` 或走 `deploy.ps1`。

Redis key：`baidu:tongji:access_token`，TTL **25 天**（与 Nest `TONGJI_TOKEN_REDIS_KEY` 一致）。

## 接口一览

| 方法 | 路径 | 鉴权 | 说明 |
|------|------|------|------|
| GET | `/api/v1/resources/baidutongji` | JWT+RBAC | query 透传 `url`（须 `/rest/2.0/tongji/` 前缀）及百度 API 参数 |
| POST | `/api/v1/scheduled-task/cache/refresh-tongji-token` | JWT **超管** | 清除缓存并 refresh，返回 `{ refreshed: true }` |

### 安全

- **SSRF 防护**：`url` 必须 `strings.HasPrefix(url, "/rest/2.0/tongji/")`
- 配置缺失时错误含「OAuth 配置不完整」
- token 过期（百度 error_code 110/111）或 Redis miss 时自动 refresh 后重试

## 与 NestJS 差异

| 项 | 说明 |
|----|------|
| 日志 | 未复刻 Nest debug 级 token 前缀日志；响应不泄露 `client_secret` |
| RBAC | dev 环境未登记 privilege 时，已登录用户可访问；生产须在 `x_privilege` 登记或超管调用 |
| 定时 refresh cron | 未实现（token 25 天 TTL + 手动 refresh 足够，与计划一致） |
