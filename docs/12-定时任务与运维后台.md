# Plan 12：定时任务与运维后台

> 对应计划：[`.cursor/plans/12-定时任务与运维后台.md`](../.cursor/plans/12-定时任务与运维后台.md)
>
> **交付日期**：2026-07-03  
> **架构形态**：4 微服务（blog-service 承载；gateway 代理 `/scheduled-task/*`）

## 交付摘要

迁移 Nest `scheduled-task` 模块：admin REST API、8 个内置 cron 种子任务、Redis 分布式锁 + 本地互斥的手动触发、执行日志、数据库备份列表/下载、超管清 RBAC 缓存。邮件类 job 经 **user-service gRPC `SendSystemEmail`** 发送（与验证码共用 SMTP 配置）。

## 目录与模块

| 路径 | 职责 |
|------|------|
| `services/blog/internal/scheduledtask/` | 任务 CRUD、日志、种子 Sync、备份路径校验 |
| `services/blog/internal/scheduledtask/jobs/` | 8 个业务 job |
| `services/blog/internal/blog/scheduler/` | robfig/cron 动态注册/注销 |
| `services/blog/internal/crossdb/` | 共享库直查（operation_log、sensitive_word_hit） |
| `services/blog/internal/handler/scheduled_task_handler.go` | HTTP 路由 |
| `proto/user/v1/user.proto` | 新增 `SendSystemEmail` RPC |
| `pkg/usersvc/email_grpc.go` | blog → user 系统邮件客户端 |
| `deploy/postman/scheduled-task-smoke.json` | Newman 冒烟 |

### 8 个种子任务

| name | cron | 说明 |
|------|------|------|
| `daily_interaction_notify` | `0 0 10 * * *` | 每日互动邮件 |
| `scheduled_publish` | `0 * * * * *` | 定时文章发布 + `blog.article.published` |
| `monthly_report` | `0 0 10 1 * *` | 月报邮件 |
| `link_health_check` | `0 0 3 * * 1` | 友链探测 + 邮件 |
| `expired_data_cleanup` | `0 30 2 * * *` | 清理 operation_log / task_log |
| `database_backup` | `0 0 4 * * *` | mysqldump |
| `draft_cleanup` | `0 0 5 * * *` | 90 天草稿软删 |
| `sensitive_word_alert` | `0 0 9 * * *` | pending 命中告警邮件 |

## 配置与环境

```yaml
# configs/blog.yaml / user.yaml 共用
app:
  notify_email: "admin@example.com"   # 系统邮件收件人（Nest app_notifyEmail）

mail:
  host: smtp.example.com
  port: 465
  user: ...
  pass: ...

backup:
  dir: ""                  # 默认 {cwd}/backups
  mysqldump_path: ""       # 默认 mysqldump；Windows 需配置完整路径
```

blog-service 依赖 `grpc.user_addr` 调用 user gRPC 发信。

## 本地启动与验收

```powershell
# 四进程（gateway 8000）
make dev-all

# 超管 token
go run scripts/dev_login.go --token-only

# Newman 冒烟
newman run deploy/postman/scheduled-task-smoke.json `
  --env-var baseUrl=http://127.0.0.1:8000 `
  --env-var token=$ADMIN_TOKEN

# 手动触发定时发布
curl -sf -X POST http://127.0.0.1:8000/api/v1/scheduled-task/trigger/scheduled_publish `
  -H "Authorization: Bearer $ADMIN_TOKEN"
```

单元测试：

```powershell
go test ./services/blog/internal/scheduledtask/... ./services/blog/internal/blog/scheduler/... -count=1
```

## 与 NestJS 差异

| 项 | 说明 |
|----|------|
| 邮件 | Go 经 gRPC 调 user-service，Nest 进程内直连 EmailService |
| `refresh-tongji-token` | Plan 16 已实现，委托 `ResourcesService.ForceRefreshTongjiAccessToken` |
| 多实例 cron | MVP 单实例；Redis 锁仅保证 trigger/定时执行互斥，未做 cron _leader 选举 |
| Windows 备份 | 需配置 `backup.mysqldump_path` |
