# Plan 01：脚手架与公共基础

> 对应计划：[`.cursor/plans/01-脚手架与公共基础.md`](../.cursor/plans/01-脚手架与公共基础.md)
>
> **交付日期**：2026-06-30  
> **架构形态**：模块化单体（`services/monolith`）

## 交付摘要

初始化 `blog-server-go` monorepo：Hertz + Ent + rueidis + wire + Viper + zap 骨架可本地启动；Ent 全量 schema 自 Nest `myblog` 库反向生成；`GET /health` 与 `GET /api/v1/health` 返回与 Nest 一致的 `{code, message, data}` 包装。

## 目录与模块

```
blog-server-go/
├── proto/                          # user/blog/rpg proto 占位（未启 gRPC）
├── pkg/
│   ├── config/                     # Viper 加载 configs/monolith.yaml
│   ├── response/                   # 统一 JSON 响应体
│   ├── errcode/                    # 业务错误码
│   ├── crypto/                     # bcrypt + 旧 PBKDF2 兼容
│   ├── logger/                     # zap
│   └── timeutil/
├── services/monolith/
│   ├── cmd/main.go                 # 入口，CONFIG_PATH 可覆盖配置
│   ├── ent/schema/                 # 全量 Ent schema（user/blog/rpg 表）
│   ├── internal/
│   │   ├── app/                    # wire 装配 + 生命周期
│   │   ├── data/                   # Ent / Redis 客户端
│   │   ├── handler/                # health 路由
│   │   ├── middleware/             # Recovery、RequestID、Logger、CORS
│   │   └── server/                 # Hertz 实例与中间件注册
├── configs/monolith.yaml
├── deploy/sql/local/               # 本地 x_my_blog 初始化 SQL
├── scripts/                        # ent schema 生成、bootstrap
├── buf.yaml / buf.gen.yaml
└── Makefile
```

| 路径 | 职责 |
|------|------|
| `services/monolith/internal/app/wire.go` | wire 注入 Config、Logger、Ent、Redis、Hertz、Handler |
| `services/monolith/internal/data/data.go` | 启动时探测 `user` 表与 Redis PING |
| `services/monolith/internal/handler/health.go` | `/health`、`/api/v1/health` |
| `scripts/gen_ent_schema.go` | 从 MySQL 反向生成 Ent schema |
| `scripts/bootstrap_x_my_blog.go` | 克隆 `myblog` → `x_my_blog` 供本地开发 |

## 配置与环境

| 项 | 说明 |
|----|------|
| `CONFIG_PATH` | 可选，默认 `configs/monolith.yaml` |
| `app.api_prefix` | `/api/v1`，与 Nest 一致 |
| `http.addr` | `:5000`（与 Nest 开发端口相同，二选一启动） |
| `mysql.database` | 本地开发库 `x_my_blog`，表前缀 `x_` |
| `mysql.schema_source_database` | Ent 生成回退源库 `myblog` |
| `redis.db` | `1`（与 Nest `.env` 对齐） |

首次本地库初始化：

1. **MySQL 管理员**先建库授权（`jiangxia` 无全局 `CREATE DATABASE` 权限时必做）：

```powershell
mysql -u root -p < deploy/sql/local/001_create_x_my_blog.sql
```

2. 克隆表结构并生成 Ent（需 Nest 源库 `myblog` 已有表；可先启动一次 Nest 让 TypeORM 同步）：

```powershell
# Windows（无 make 时用）
.\scripts\bootstrap-db.ps1

# Linux / macOS / Git Bash
make bootstrap-db
```

等价于依次执行：`bootstrap_x_my_blog.go` → `gen_ent_schema.go` → `ent-gen`。

## 接口一览

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/health` | 健康检查，`data: "ok"` |
| GET | `/api/v1/health` | 同上，与 Nest 路径一致 |

响应示例：

```json
{"code":200,"bizCode":200,"message":"success","data":"ok"}
```

HTTP 状态码恒为 **200**；业务成败看 JSON `code`（与 Nest `TransformInterceptor` 一致）。

## 本地启动与验收

```powershell
# 依赖：MySQL（3306）、Redis（6379）须已运行

# Windows（无 make 时用）
.\scripts\dev.ps1

# Linux / macOS / Git Bash
make dev
```

等价命令（PowerShell）：

```powershell
$env:CONFIG_PATH = "configs/monolith.yaml"
go run ./services/monolith/cmd/main.go
```

冒烟（PowerShell）：

```powershell
(Invoke-RestMethod http://localhost:5000/health).code -eq 200
(Invoke-RestMethod http://localhost:5000/api/v1/health).code -eq 200
```

生成物可重复构建（Windows 分步执行）：

```powershell
go run github.com/google/wire/cmd/wire@latest ./services/monolith/internal/app
cd services/monolith/ent; go generate ./...
```

启动日志应出现 `ent client connected` 与 `redis client connected`。

## 与 NestJS 差异

| 项 | 说明 |
|----|------|
| 密码哈希 | `pkg/crypto` 已实现 bcrypt + 旧 PBKDF2 校验与静默升级逻辑，本阶段未暴露登录接口 |
| 迁移工具 | 本地用 `bootstrap-db` 克隆库，非 golang-migrate 增量脚本（与现网 schema 一致时足够） |
| gRPC | proto 仅占位，运行时未启动 |
