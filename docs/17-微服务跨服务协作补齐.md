## 交付摘要

补齐 Plan 11 遗留的 **blog ↔ user ↔ rpg** 跨服务 stub：敏感词过滤、文章数据权限、C 端作者过滤、部门名回显、RPG 命中记录分页、审核后实体状态同步（F 项与 Plan 13 已联通）。微服务模式下行为与 Nest 单体「同进程调 service」等价。

## 目录与模块

| 路径 | 职责 |
|------|------|
| `proto/user/v1/user.proto` | 扩展 7 个 RPC + `GetUserResponse.dept_id` |
| `services/user/internal/user/grpcserver/cross.go` | 敏感词/部门/数据权限/命中记录 gRPC 实现 |

| `pkg/usersvc/port.go` | `CrossClient` 聚合端口 |

| `services/rpg/internal/app/adapters.go` | `hitRecordsAdapter` 调 user gRPC |

## 配置与环境

无新增配置项；沿用各服务 yaml 中 `grpc.user_addr` / `grpc.blog_addr`：

```yaml
# configs/blog.yaml / configs/rpg.yaml
grpc:
  user_addr: "127.0.0.1:50052"

# configs/user.yaml
grpc:
  addr: ":50052"
  blog_addr: "127.0.0.1:50051"   # 敏感词审核后同步 comment 状态
```

## 接口一览

### 新增 user.v1.UserService gRPC（`:50052`）

| RPC | 调用方 | 说明 |
|-----|--------|------|
| EvaluateContent | blog-service | 敏感词分级检测 |
| CreateHitRecord | blog-service | 写入命中记录 |
| ListActiveUserIDs | blog-service | C 端过滤锁定作者 |
| GetDept | blog-service | 文章列表/详情 `deptName` |
| ResolveAccessibleDeptIDs | blog-service | admin 文章数据权限 |
| AssertDeptAccess | blog-service | 编辑/删除文章机构校验 |
| ListSensitiveWordHits | rpg-service | `GET /rpg/hit-records` 数据源 |

## 与 NestJS 差异

- user gRPC 不可用时 blog **fail-closed**（返回错误，不静默放行），与 Nest 单体「敏感词服务必达」一致。
- `ListSensitiveWordHits` gRPC 返回 map 结构，RPG HTTP 层与单体字段对齐（`list` + `pagination`）。
