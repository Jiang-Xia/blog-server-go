# Plan 04：敏感词与运维骨架

> 对应计划：[`.cursor/plans/04-敏感词与运维骨架.md`](../.cursor/plans/04-敏感词与运维骨架.md)
>
> **交付日期**：2026-06-30  
> **架构形态**：模块化单体（`internal/user/sensitive` + `internal/blog/{notification,operationlog,scheduler}`）

## 交付摘要

迁移 Nest 敏感词 CRUD/过滤、站内通知 HTTP、操作日志记录与 cron 框架骨架至 Go 单体。路径与 Nest 对齐（`/sensitive-word`、`/notification`、`/operation-log`）；过滤接口 `sensitive.FilterService` 已导出供 Plan 06/07 comment/msgboard 注入。

## 目录与模块

| 路径 | 职责 |
|------|------|
| `services/monolith/internal/user/sensitive/` | 敏感词 CRUD、命中审核、`FilterService` 过滤接口 |
| `services/monolith/internal/blog/notification/` | 通知列表/未读/已读、`since` 骨架 |
| `services/monolith/internal/blog/operationlog/` | 操作日志写入与查询/清理 |
| `services/monolith/internal/blog/scheduler/` | `robfig/cron/v3` 占位 job |
| `services/monolith/internal/middleware/operation_log.go` | 写操作异步记录（对齐 Nest Interceptor） |
| `services/monolith/internal/handler/sensitive_handler.go` 等 | HTTP 端点 |
| `deploy/postman/ops-smoke.json` | M2 运维冒烟 |

## 配置与环境

无新增配置项；复用 `configs/monolith.yaml` 中 MySQL/Redis/JWT。cron 在应用 `Run()` 时自动启动占位 job。

## 接口一览

### `/api/v1/sensitive-word/*`（Nest SensitiveWordController）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/sensitive-word` | 分页列表（keyword/category/status） |
| POST | `/sensitive-word` | 新增 |
| POST | `/sensitive-word/batch` | 批量导入 |
| PATCH | `/sensitive-word/:id` | 更新 |
| DELETE | `/sensitive-word/:id` | 删除 |
| GET | `/sensitive-word/hit` | 命中记录分页 |
| POST | `/sensitive-word/hit/:id/approve` | 审核通过 |
| POST | `/sensitive-word/hit/:id/reject` | 审核拒绝 |

### `/api/v1/notification/*`（Nest + since 骨架）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/notification/list` | 当前用户通知分页 |
| GET | `/notification/unread-count` | 未读数 |
| PATCH | `/notification/read` | 标记已读（`?id=` 可选） |
| GET | `/notification/since` | `?seq=` 增量拉取（HTTP 骨架，Plan 08 WS 补漏） |

### `/api/v1/operation-log/*`（Nest OperationLogController）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/operation-log` | 分页查询（module/action/username/keyword） |
| DELETE | `/operation-log/clean` | 清理（`?days=` 可选） |

## 与 NestJS 差异

| 项 | 说明 |
|----|------|
| 路径前缀 | 计划文档写 `/admin/sensitive-word`，实际与 Nest 一致为 `/sensitive-word`（无 admin 前缀） |
| 敏感词种子 | Nest `onModuleInit` 自动导入种子；Go 未内置种子，由现网数据或 API 创建 |
| 命中审核联动 | Nest 会同步 comment/msgboard/reply 状态；Go 仅更新 hit 记录（实体模块在 Plan 06/07） |
| notification since | Nest 无此接口；Go 新增 HTTP 骨架，以 `id` 作 seq |
| scheduled-task | Nest 含 8 个业务 cron；Go 仅占位 job，业务任务在 Plan 09 |
| Ent schema 与现网 | `operation_log`/`site_notification`/`sensitive_word_hit` 无 TimeMixin 全字段；已按 Nest 表结构修正 schema 并 `ent generate` |
