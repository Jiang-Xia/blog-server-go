## 交付摘要

blog-service / monolith 业务写路径已向 Redis Stream `blog:events` 发布领域事件，与 Plan 09 已接入的 rpg-service 消费者闭环。发文章、评论、点赞、收藏、阅读等行为不再依赖 `/dev/event-publish` 调试接口即可驱动 RPG 经验与任务进度；敏感词命中（登录用户 + hpPenalty）会发布 `blog.sensitive-word.hit`。

## 配置与环境

无新增配置；沿用各服务 Redis 与 `blog:events` Stream（Plan 08）。

## 接口一览

无新增 REST/gRPC 接口。变更均为既有写路径的副作用（Stream 发布）。

| 事件 type | 发布时机 |
|-----------|----------|
| `blog.article.published` | 创建/编辑/定时任务发布 |
| `blog.article.updated` | 已发布文章编辑 |
| `blog.article.unpublished` | 下架 |
| `blog.article.deleted` | 软删或物理删 |
| `blog.article.viewed` | POST `/article/views` 成功 |
| `blog.comment.created` | 评论入库 |
| `blog.reply.created` | 回复入库 |
| `blog.msgboard.created` | 留言入库（有 uid 时） |
| `blog.like.created` | 登录用户点赞 |
| `blog.collect.created` | 收藏 |
| `blog.sensitive-word.hit` | 命中且 uid>0 且 hpPenalty>0（拒绝前） |

## 本地启动与验收

```powershell
# 微服务全栈
make dev-all

# 登录取 token
go run scripts/dev_login.go --token-only

# 发评论 → 查 RPG 经验
curl -sf -X POST http://127.0.0.1:8000/api/v1/comment/create `
  -H "Authorization: Bearer $TOKEN" `
  -H "Content-Type: application/json" `
  -d '{"articleId":1,"content":"测试评论"}'

curl -sf http://127.0.0.1:8000/api/v1/rpg/status -H "Authorization: Bearer $TOKEN"

# 单元测试
go test ./services/blog/internal/blog/service/... -count=1

# 集成测试（需 dev-all 运行中）
go test -tags=integration ./test/integration/... -run TestIntegrationCommentPublishesRPGExp -count=1
```

## 与 NestJS 差异

| 项 | Nest | Go |
|----|------|-----|
| Stream MAXLEN | `~10000` | 当前 XADD 未设 MAXLEN（Plan 08 常量已定义，后续可补） |
| 发布失败 | catch 日志 | `Publisher.Publish` 只打 Warn，不阻断主流程 |
| msgboard uid | 可选 `(msgboard as any).uid` | 从请求 body `uid` 字段读取 |
| article.viewed viewerUid | 可选 | 当前未传 viewerUid（RPG 侧 dedup 用 0） |
