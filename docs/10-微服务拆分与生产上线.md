# Plan 10：微服务拆分与生产上线

> 对应计划：[`.cursor/plans/10-微服务拆分与生产上线.md`](../.cursor/plans/10-微服务拆分与生产上线.md)
>
> **交付日期**：2026-06-30  
> **架构形态**：4 微服务 + 共享 MySQL/Redis

## 交付摘要

将 Plan 01–09 的模块化单体物理拆分为 **gateway / user / blog / rpg** 四进程：gateway 在 `:8000` 提供统一 REST 入口与 JWT 透传；user-service 额外暴露 gRPC（`:50052`）供 blog/rpg 跨服务读用户；blog-service 保留 WebSocket Hub；docker-compose 一键拉起全栈 + Jaeger。单体模式（`:5000`）仍可用，便于本地对照与回滚。

## 目录与模块

| 路径 | 职责 |
|------|------|
| `services/gateway/cmd/` | API Gateway 入口（REST BFF + 反向代理） |
| `services/gateway/internal/proxy/` | 路径 → user/blog/rpg 转发 |
| `services/gateway/internal/aggregator/` | BFF 聚合（`/pub/stats`） |
| `services/gateway/internal/middleware/` | JWT 验签 → `x-user-id` 透传 |
| `services/monolith/cmd/user|blog|rpg/` | 三内部服务入口（复用 monolith wire） |
| `services/monolith/internal/handler/register_*.go` | 按域拆分路由注册 |
| `services/monolith/internal/user/grpcserver/` | user gRPC 服务端 |
| `services/monolith/internal/user/grpc_client.go` | blog/rpg 侧 UserService gRPC 客户端 |
| `proto/user/v1/user.proto` | GetUser / GetUserBatch / VerifyToken |
| `pkg/config/` | ServiceMode、GRPC、Proxy、Observability |
| `pkg/jwtauth/` | JWT 签发/校验（gateway 与 monolith 共享） |
| `pkg/grpcmeta/` | gRPC metadata 与 Auth 拦截器 |
| `pkg/otel/`、`pkg/metrics/` | OpenTelemetry / Prometheus |
| `configs/{user,blog,rpg,gateway}.yaml` | 本地开发分服务配置 |
| `configs/docker/*.yaml` | docker-compose 内网主机名配置 |
| `deploy/docker/docker-compose.yml` | 4 服务 + MySQL + Redis + Jaeger |
| `deploy/docker/mysql.cnf` | 2G 机器 MySQL 调优 |
| `services/*/Dockerfile` | multi-stage + distroless |

## 配置与环境

### ServiceMode

| 值 | 说明 |
|----|------|
| `monolith`（默认） | 单体，全部路由 + 背景任务 |
| `user` | 用户/RBAC/验证码/敏感词/操作日志 + gRPC |
| `blog` | 文章/互动/资源/通知/WS |
| `rpg` | RPG/支付/公开主页 |
| `gateway` | 仅代理与 BFF，无 DB |

### 关键配置项

```yaml
app:
  service_mode: blog   # user | blog | rpg | gateway | monolith
grpc:
  addr: ":50052"       # user-service gRPC 监听
  user_addr: "127.0.0.1:50052"  # blog/rpg 连接 user gRPC
proxy:
  user_url: "http://127.0.0.1:5002"
  blog_url: "http://127.0.0.1:5001"
  rpg_url: "http://127.0.0.1:5003"
observability:
  enable_metrics: true
  enable_pprof: true
  otlp_endpoint: "jaeger:4317"   # Docker 内 Jaeger
```

Ent MySQL 连接池：`MaxOpenConns=15`（4×15=60 < MySQL 100）。

## 接口一览

对外 REST 路径与 Nest/单体一致，统一经 gateway `:8000` 访问：

| 域 | 示例路径 | 后端 |
|----|----------|------|
| 用户/管理 | `/api/v1/user/*`、`/role`、`/admin/menu`、`/captcha` | user-service |
| 博客 | `/api/v1/article/*`、`/comment`、`/notification` | blog-service |
| WS | `/realtime` | blog-service |
| RPG/支付 | `/api/v1/rpg/*`、`/pay/*`、`/user/public/*` | rpg-service |
| BFF | `/api/v1/pub/stats` | gateway 聚合 |

gRPC（内部）：

| 服务 | 方法 |
|------|------|
| user.v1.UserService | GetUser、GetUserBatch、VerifyToken |

## 本地启动与验收

### 方式 A：仍用单体（对照）

```bash
make dev
curl -sf http://localhost:5000/api/v1/health
```

### 方式 B：四服务本地联调

终端分别启动（需本机 MySQL + Redis 已就绪）：

```bash
make dev-user    # :5002 + gRPC :50052
make dev-blog    # :5001
make dev-rpg     # :5003
make dev-gateway # :8000
```

```bash
curl -sf http://localhost:8000/api/v1/health
curl -sf http://localhost:8000/api/v1/pub/stats
# 域冒烟（与 Plan 01–09 相同集合，baseUrl 改为 8000）
# deploy/postman/*.json + --env-var baseUrl=http://localhost:8000
```

### 方式 C：docker-compose

```bash
make up
curl -sf http://localhost:8000/api/v1/health
docker stats --no-stream   # 验证 2G 内存预算
# Jaeger UI: http://localhost:16686
make down
```

### 构建

```bash
make proto   # buf generate
make wire
make build   # bin/gateway、bin/user、bin/blog、bin/rpg、bin/monolith
```

## 与 NestJS 差异

- 生产切流目标为 **gateway :8000**，Nest `:5000` 下线前需 Nginx 改 upstream（见运维 checklist）。
- 微服务模式下 blog/rpg 通过 **gRPC** 读用户摘要，不再本地 JOIN user 表（与 Plan 05 端口设计一致）。
- `/pub/stats` 在 gateway BFF 聚合；article 计数仍为 mock（与单体 pub 一致）。
