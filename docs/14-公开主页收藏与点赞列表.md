> 对应计划：[`.cursor/plans/14-公开主页收藏与点赞列表.md`](../.cursor/plans/14-公开主页收藏与点赞列表.md)
>
> **交付日期**：2026-07-03  
> **架构形态**：4 微服务（rpg HTTP → blog gRPC）

## 交付摘要

实现 `GET /api/v1/user/public/:uid/collects` 与 `/likes` 的真实分页数据，替换 rpg-service 空列表 stub。blog-service 经 gRPC 提供收藏/点赞联表查询（对齐 Nest 过滤规则与 `mapPublicArticle` 字段），rpg-service 通过 `blog_addr` 调用。

## 接口一览

### HTTP（gateway → rpg-service）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/user/public/:uid/collects` | Query：`page`（默认 1）、`pageSize`（默认 10） |
| GET | `/api/v1/user/public/:uid/likes` | 同上 |

响应 `data`：`{ list: [...], pagination: { total, page, pageSize, pages } }`

### gRPC（rpg → blog）

| RPC | 说明 |
|-----|------|
| `blog.v1.ArticleService/ListPublicCollectArticles` | 收藏列表，按 `col.createTime DESC` |
| `blog.v1.ArticleService/ListPublicLikeArticles` | 点赞列表（`like.status=1`），按 `like.id DESC` |

过滤：文章 `isDelete=false` 且 `status=publish`；作者 `status=active` 且 `isDelete=false`。

## 与 NestJS 差异

- `createTime` 序列化为 RFC3339Nano 字符串（Nest 为 ISO Date，前端均可解析）。
- 点赞数使用 `article.likes` 字段（Nest 优先联查 `articleLikes` 计数；业务上经 `SyncArticleLikes` 保持一致）。
- `GET /user/public/:uid/articles` 仍为简化实现（无分页），本计划范围外。

## 已知限制与后续

- 作者过滤经共享库 `x_user` JOIN（与 Nest SQL 一致）；Plan 17 可改为纯 gRPC `GetUserBatch` 二次过滤。
- 公开主页主卡片 `GET /user/public/:uid` 仍由 gateway BFF-gRPC 处理，未变更。
