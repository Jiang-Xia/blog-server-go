# Plan 11：微服务代码物理拆分

> 对应计划：[`.cursor/plans/11-微服务代码物理拆分.md`](../.cursor/plans/11-微服务代码物理拆分.md)
>
> **交付日期**：2026-07-01  
> **架构形态**：4 微服务（user/blog/rpg 已物理拆分）+ 共享 MySQL/Redis

## 交付摘要

Plan 10 的运行时四进程拆分已补完为**代码物理拆分（主体完成）**：**user / blog / rpg** 三服务均拥有独立 `internal/`、`ent/schema/`、`wire` 与 `cmd`；跨服务用户读统一经 `pkg/usersvc` gRPC。gateway 实现 P0 gRPC BFF（`pub/stats`、`article/info`）并修复 `/health` 统一 `{code,message,data}`。monolith 保留 user/blog/rpg 副本供 `make dev` 单体回滚。

## 目录与模块

| 路径 | 职责 |
|------|------|
| `services/user/cmd/` | user-service 入口 |
| `services/user/ent/schema/` | user 域 13 表 + mixin |
| `services/user/internal/` | auth/RBAC/敏感词/操作日志/gRPC server |
| `services/blog/cmd/` | blog-service 入口 |
| `services/blog/ent/schema/` | blog 域 16 表 + mixin |
| `services/blog/internal/` | 文章/互动/WS/scheduler/event |
| `services/blog/internal/blog/grpcserver/` | ArticleService gRPC |
| `services/rpg/cmd/` | rpg-service 入口（替代 `monolith/cmd/rpg`） |
| `services/rpg/ent/schema/` | rpg 域 21 schema + pay_order + mixin |
| `services/rpg/internal/` | rpg/pay/event/handler + userport/articleport/wspush |
| `pkg/usersvc/` | 跨服务 UserDTO + gRPC 客户端 |
| `services/gateway/internal/grpcclient/` | 连 user/blog gRPC |
| `services/gateway/internal/aggregator/` | pub/stats、article/info BFF |
| `services/monolith/` | **Nest 替换主入口**（Plan 22），全量业务 + RAG + 定时任务 |

### Ent 表归属

| 服务 | schema 文件 |
|------|-------------|
| user | user, role, dept, menu, privilege, RBAC 关联表, sensitive_word*, operation_log, mixin |
| blog | article, category, tag, article_tags_tag, comment, reply, like, collect, msgboard, link, file, my_file, site_notification, scheduled_task*, mixin |
| rpg | rpg_* , pay_order, mixin |
| monolith（遗留） | 全量 50 表，含 RAG 三表 |

## 接口与 BFF

| 路径 | 实现 |
|------|------|
| `GET /api/v1/health`（gateway） | 统一 `{code:200,message,data:"ok"}` |
| `GET /api/v1/pub/stats` | gRPC：`blog.GetPubStats` + `user.CountUsers` |
| `GET /api/v1/article/info` | gRPC：`blog.GetArticleDetail` |
| `GET /api/v1/user/public/:uid` | gRPC：`rpg.GetPublicProfile` |
| `GET /api/v1/user/public/:uid/articles` 等 | HTTP 代理 → rpg-service |
| 其余 REST | HTTP 反向代理（Plan 10） |
| 路由全表 | [api-routes.md](./api-routes.md) |
| `/realtime` WebSocket | 代理至 blog-service |

## 与 Plan 10 差异

| 项 | Plan 10 | Plan 11 |
|----|---------|---------|
| 业务代码 | `monolith/internal` | `services/{user,blog,rpg}/internal` |
| Ent | 单包 50 schema | 各服务生成子集 client |

| gateway BFF | HTTP 聚合/mock | P0 走 gRPC |

## 已知限制

1. **迁移窗口双份代码**：monolith 与各微服务各有一份实现；**Plan 22 后以 monolith 为功能基准**，微服务不强制同步。
2. **Go internal 规则**：monolith 不能 import `services/*/internal`，故保留双份代码；微服务目录用于学习拆分部署。
3. **blog 跨域 stub**：敏感词 Noop、部门 ACL 宽松、ListActiveUserIDs 等待 user gRPC 扩展。
4. **rpg Ent mixin**：rpg/pay 表仅有 `createTime/updateTime`，schema 使用 `TimestampMixin`（非 `TimeMixin`）；repo 不做 `isDelete` 软删过滤。
5. **hit-records**：rpg-service 返回空列表（user 域表不在 rpg ent）。
6. **monolith**：Plan 22 起为 **Nest 替换主入口**（见 [`services/monolith/README.md`](../services/monolith/README.md)）；微服务目录保留作架构学习。
