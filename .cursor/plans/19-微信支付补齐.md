# Plan 19：微信支付补齐（可选）

## 元信息

| 项 | 内容 |
|----|------|
| **目标** | 对齐 Nest 微信支付能力：H5 拉起小程序、openid、预下单/回调（与现有支付宝并列） |
| **前置依赖** | [09-RPG与支付.md](./09-RPG与支付.md) 支付宝链路已通；[11-微服务代码物理拆分.md](./11-微服务代码物理拆分.md) |
| **周期** | ~1–2 周（视 SDK 选型与沙箱联调） |
| **架构形态** | rpg-service `/pay/*` |
| **里程碑** | M6 Nest 差异补齐（**可选**） |
| **Nest 对照** | `modules/pay/pay.service.ts` 微信分支 |

## 背景

Go 侧 `PayService` 仅集成 **smartwalle/alipay**：

- `POST /pay/h5-open-mini` 在 `type=wechat` 时返回 `wechat h5 open mini not implemented`
- `POST /pay/notice` **仅**处理支付宝异步通知
- `POST /pay/openid` 当前走支付宝 oauth

若 **blog-home-uniapp 微信小程序充值** 必须走微信 JSAPI/小程序支付，需本计划；若生产仅用支付宝，可延后。

## 模块范围

| 能力 | Nest | Go 目标 |
|------|------|---------|
| H5 拉起微信小程序 | `buildWechatMiniProgramUrl` + access_token | 同等 REST 调微信 API |
| code → openid | 微信 `jscode2session` | `/pay/openid` 支持 channel=wechat |
| 预下单 | 微信 JSAPI/小程序下单 | `/pay/trade/create` channel=wechat |
| 支付回调 | 微信 notify XML/JSON | `/pay/notice` 按 channel 分发 |
| 本地订单 | `pay_order.channel=wechat` | 已有常量，补业务逻辑 |

配置段（`configs/*.yaml` → `pay`）：

- `wechat_app_id`、`wechat_secret`（已有字段名对照 Nest `wechatAppId`）
- `wechat_mch_id`、`wechat_api_key` / 证书（V3 API）
