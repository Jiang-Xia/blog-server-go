## 关键实现要点

### JWT 认证（单体版，Plan 10 前 gateway 暂不独立）

```go
// internal/user/auth/jwt.go
// 签发 access + refresh token
// 中间件：解析 Bearer token → ctx 写入 userID + roles
```

- access TTL / refresh TTL 与 NestJS 配置一致
- GitHub OAuth：`golang.org/x/oauth2`
- 验证码：`mojocn/base64Captcha`，Redis 存 session

### 模块间 interface（为 Plan 10 gRPC 做准备）

```go
// internal/user/port.go
type UserService interface {
    GetUser(ctx context.Context, id uint64) (*UserDTO, error)
    GetUserBatch(ctx context.Context, ids []uint64) ([]*UserDTO, error)
}
```

Plan 05 起 article 等模块依赖此 interface 的**本地实现**。

### 密码兼容

- 旧数据：crypto-js / node-jsencrypt 哈希 — 登录验证通过后静默升级 bcrypt
- 新注册：直接 bcrypt
