# Plan 06：博客互动（模块化单体）

## 元信息

| 项 | 内容 |
|----|------|
| **目标** | 迁移 comment、reply、like、collect 模块，评论/点赞/收藏全链路可用 |
| **前置依赖** | [05-文章内容.md](./05-文章内容.md) 验收通过 |
| **周期** | ~2 周 |
| **架构形态** | 模块化单体（`internal/blog/` 包） |
| **里程碑** | M3 博客域 |
| **原方案章节** | 三（3.1 blog-service、3.3 表归属）、五（5.6 Ent）、8.3 第5-6周 |

## 模块范围

| NestJS 路径 |
|-------------|
| `features/comment/`、`reply/`、`like/`、`collect/` |

### 负责表

`comment`, `reply`, `like`, `collect`

## API 路径（保持兼容）

| 路径 | 说明 |
|------|------|
| `/api/v1/comment/*` | 评论 |
| `/api/v1/reply/*` | 回复 |
| `/api/v1/like/*` | 点赞 |
| `/api/v1/collect/*` | 收藏 |

## 关键实现要点

### 敏感词过滤

- comment 创建/更新时调用 Plan 04 的敏感词过滤函数
- 命中策略与 NestJS 一致

### 幂等

- like/collect：数据库唯一索引 + 幂等处理
- 取消操作与 NestJS 行为一致

## 风险与注意点

| 风险 | 对策 |
|------|------|
| 点赞/收藏并发 | 唯一索引 + ON DUPLICATE KEY 或事务 |
