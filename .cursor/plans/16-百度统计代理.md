# Plan 16：百度统计代理

## 元信息

| 项 | 内容 |
|----|------|
| **目标** | 迁移 `GET /resources/baidutongji` OAuth 代理 + token 缓存；对接 Plan 12 `refresh-tongji-token` |
| **前置依赖** | [12-定时任务与运维后台.md](./12-定时任务与运维后台.md) 验收通过（共享 refresh 入口）；与 Plan 15 无硬依赖 |
| **周期** | ~3–5 天 |
| **架构形态** | 4 微服务（**blog-service** `resources` 包扩展） |
| **里程碑** | M6 Nest 差异补齐 |
| **Nest 对照** | `features/resources/resources.service.ts` + `resources.controller.ts` |

## 背景

Plan 07 交付文档注明 **`/resources/baidutongji` 未迁移**。blog-admin 仪表盘与 Nest `scheduled-task` 超管接口 `POST /scheduled-task/cache/refresh-tongji-token` 依赖百度 OpenAPI token 生命周期管理。

## 模块范围

| 能力 | 说明 |
|------|------|
| Token 刷新 | `refresh_token` → `access_token`，写 Redis + 进程内缓存 |
| 代理请求 | 客户端传 `url`（必须以 `/rest/2.0/tongji/` 开头）+ 其它 query → 服务端附加 `access_token` 转发 |
| 手动刷新 | `POST /scheduled-task/cache/refresh-tongji-token`（Plan 12）调用同一 `RefreshAccessToken` |
| 配置校验 | 缺配置时返回明确错误（对齐 Nest HttpException 文案） |

### 配置（`app` 或 `resources` 段）

| 键 | Nest env 对照 |
|----|----------------|
| `tongji_refresh_token` | `app_tongjiRefreshToken` |
| `tongji_client_id` | `app_tongjiClientId` |
| `tongji_client_secret` | `app_tongjiClientSecret` |

Redis key：对齐 Nest `TONGJI_TOKEN_REDIS_KEY`（交付文档写死常量名）；TTL **25 天**。

## API 路径

| 方法 | 路径 | 鉴权 | 说明 |
|------|------|------|------|
| GET | `/api/v1/resources/baidutongji` | RBAC（admin 仪表盘） | query 透传 `url` + 百度 API 参数 |
| POST | `/api/v1/scheduled-task/cache/refresh-tongji-token` | **超管** | Plan 12 已实现路由时委托 resources 服务 |

### 安全

- **url 白名单**：必须 `strings.HasPrefix(url, "/rest/2.0/tongji/")`（防 SSRF）
- 日志中 token 仅打印前 10 位 + `***`
- 响应不泄露 `client_secret`

## 任务清单

- [ ] `ResourcesService`（或 `BaiduTongjiService`）实现 `RefreshAccessToken`、`GetTongjiData`
- [ ] HTTP handler：`GET /resources/baidutongji`
- [ ] Redis 读写（复用 blog-service rueidis）
- [ ] Plan 12 `refreshTongjiToken` 调用同一 service（避免双实现）
- [ ] configs 示例写入 `configs/monolith.yaml` / docker 示例
- [ ] RBAC：admin 权限 fallback 或 patch SQL 登记
- [ ] `deploy/postman/resources-smoke.json` 扩展（mock 或 skip 无配置）
- [ ] 单元测试：url 校验、token 缓存 key

## 验收标准

- [ ] 配置完整时 `refresh-tongji-token` 返回 `{ refreshed: true }`，Redis 有 access_token
- [ ] `GET /resources/baidutongji?url=/rest/2.0/tongji/...` 返回百度 JSON（或沙箱 mock）
- [ ] 非法 `url=/etc/passwd` 或 `http://evil` 被拒绝
- [ ] 配置缺失时错误 message 含「OAuth 配置不完整」类提示
- [ ] token 过期自动 refresh（先读 Redis，miss 则 refresh）

### 可脚本化验收

```bash
make dev-all
# 需配置 tongji_* 后：
curl -sf "http://127.0.0.1:8000/api/v1/resources/baidutongji?url=/rest/2.0/tongji/...&..." \
  -H "Authorization: Bearer $ADMIN_TOKEN"
```

## 本计划不做

- 百度统计 UI 组件（前端 admin 已有）
- 定时自动 refresh cron（token 25 天 TTL + 手动 refresh 足够；可选后续 job）
