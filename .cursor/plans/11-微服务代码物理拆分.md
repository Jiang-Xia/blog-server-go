# Plan 11：微服务代码物理拆分

## 元信息

| 项 | 内容 |
|----|------|
| **目标** | 将 Plan 10「运行时拆分」补完为**代码物理拆分**：各服务独立 `internal/` + Ent schema + wire；gateway HTTP 代理逐步改为 gRPC BFF |
| **前置依赖** | [10-微服务拆分与生产上线.md](./10-微服务拆分与生产上线.md) 验收通过（四进程 + docker-compose + 冒烟可用） |
| **周期** | ~3–4 周 |
| **架构形态** | 4 微服务（代码隔离）+ 共享 MySQL/Redis 单库 |
| **里程碑** | M5 深化（代码与部署一致） |
| **延续** | 总方案 8.4 物理拆分、3.3 Ent 分 schema |

## 背景：Plan 10 与 Plan 11 边界

Plan 10 已完成：

- 四进程独立部署（gateway / user / blog / rpg）
- `service_mode` 路由切分、`dev-all.ps1` 一键本地启动
- user 域 gRPC（GetUser / GetUserBatch / VerifyToken）+ blog/rpg gRPC client 雏形
- gateway HTTP 反向代理 + `/pub/stats` BFF
- docker-compose、可观测性骨架

Plan 10 **未完成**（本计划负责）：

| 缺口 | 现状 | Plan 11 目标 |
|------|------|--------------|
| 代码目录 | 业务仍在 `services/monolith/internal/` | 迁至 `services/{user,blog,rpg}/internal/` |
| wire | 四服务共用 monolith `internal/app/wire` | 每服务独立 `wire.go` + `wire_gen.go` |
| Ent | 50 schema 全在 monolith | 各服务 `ent/schema/` 只含本域表 |
| 跨服务调用 | 仅 UserService gRPC | blog/rpg proto 扩展；gateway REST→gRPC BFF |
| gateway | 全量 HTTP 代理 | 高频读路径先改 BFF，代理作 fallback |

## 阶段 A：user-service 先行（~1 周）

对照总方案 8.4，user 边界最清晰、已有 gRPC server。

| 步骤 | 工作 | 验收 |
|------|------|------|
| 1 | 创建 `services/user/internal/`，从 monolith **移动**（非 copy 留双份）`internal/user/*` | user 包 import 路径更新完毕 |
| 2 | 创建 `services/user/ent/schema/`，迁入 user 域表（见下表） | `make ent-gen-user` 可生成 |
| 3 | `services/user/internal/app/wire.go` 独立装配 | `make wire-user` 通过 |
| 4 | `services/user/cmd/main.go` 只依赖 user 模块 | 不再 import monolith/internal/app |
| 5 | 扩展 `user.proto`（VerifyToken 已有；按需 PrivilegeCheck RPC） | buf generate + breaking 通过 |
| 6 | 删除 monolith 内已迁走的 user 代码；monolith `service_mode=user` 改为调 user 二进制或暂保留 shim | user 单服务冒烟 |

### user 域 Ent schema 清单

| schema 文件 | 表 |
|-------------|-----|
| `user.go` | x_user |
| `role.go` `dept.go` `menu.go` `privilege.go` | RBAC 主表 |
| `role_users_user.go` … | RBAC 关联表 |
| `sensitive_word.go` `sensitive_word_hit.go` | 敏感词 |
| `operation_log.go` | 操作日志 |
| `mixin.go` | 公共 mixin（复制一份，不跨服务 import ent） |

**禁止**：user ent 中出现 `article` / `rpg_*` schema。

## 阶段 D：Ent 拆分策略（贯穿 A–C）

**原则：共享单库，schema 文件按服务隔离，禁止跨服务 Ent 关系。**

```
                    ┌─────────────┐
                    │  MySQL 单库  │
                    │  x_my_blog  │
                    └──────┬──────┘
           ┌───────────────┼───────────────┐
           ▼               ▼               ▼
    user/ent/client  blog/ent/client  rpg/ent/client
    (仅 user 表)     (仅 blog 表)     (仅 rpg+pay 表)
```

| 规则 | 说明 |
|------|------|
| DSN | 各服务同一 `mysql.database`，连接池仍 15/服务 |
| mixin | 各服务 `ent/schema/mixin.go` 各维护一份（字段一致） |
| 迁移 | **不**拆库；Ent migrate 仅本服务 schema 子集，生产用现有表（已存在） |
| 代码生成 | `Makefile`：`ent-gen-user` / `ent-gen-blog` / `ent-gen-rpg` |
| repo 边界 | 跨域 ID 只存 uint64，通过 gRPC/DTO 组装 |
| RAG 表 | `knowledge_chunk` 等 **不迁入** 本计划 |

### 渐进式 repo 边界（若全量搬 ent 风险大）

1. **第一步**：物理搬 Go 代码 + 各服务 ent 只 generate 本域 schema（本计划必做）
2. **第二步**（可选）：monolith ent 标记 deprecated，CI 禁止 monolith 新增 schema
3. **第三步**（可选）：删除 `services/monolith/ent/`，monolith 入口移除或仅集成测试

## 阶段 E：gateway gRPC BFF（~1 周，与 D 并行收尾）

**策略：按读多写少、聚合价值高优先迁移；写路径与文件上传暂保留 HTTP 代理。**

| 优先级 | REST 路径 | BFF 行为 | 依赖 gRPC |
|--------|-----------|----------|-----------|
| P0 | `GET /api/v1/pub/stats` | 已有 HTTP 聚合 → 改 gRPC 调 blog/user 统计 RPC | blog + user |
| P0 | `GET /api/v1/article/info` | 文章 + 作者合并 | blog.GetArticle + user.GetUser |
| P1 | `GET /api/v1/user/public/:uid` | 主页聚合 | rpg + blog + user |
| P1 | `GET /api/v1/health` | 统一 `{code,message,data}` | 本地 + 可选下游探测 |
| P2 | 列表类只读 API | 按需 BFF | blog List* |
| 保留代理 | POST/PATCH/DELETE、文件上传、`/pay/notice` | HTTP reverse proxy | — |
| 保留代理 | `/realtime` WebSocket | 直连 blog HTTP/WS | — |

### 认证

- gateway：JWT 验签（`pkg/jwtauth`）→ gRPC metadata `x-user-id` + HTTP header 双透传
- 内部服务：gRPC AuthInterceptor + HTTP middleware 从 header/metadata 取 uid，**不重复验签**

## 本计划不做

- 新业务功能 — Plan 01–09 范围外
- RAG 模块迁移（`knowledge_chunk` 等）
- MySQL 拆库 / 分库分表
- Kubernetes
- blog-home-uniapp 全量对接
- gateway 100% REST→gRPC（写路径与 WS 可长期保留 HTTP）
- NestJS 生产下线（运维阶段，与 Plan 10 相同）

## 风险与决策

| # | 风险 | 缓解 |
|---|------|------|
| 1 | 搬代码 import 雪崩 | 严格按 user→blog→rpg 顺序；每步 `go build ./...` |
| 2 | Ent 双份 mixin 漂移 | 文档约定；后续可抽 `pkg/entmixin`（非 Ent schema） |
| 3 | gRPC BFF 与 Nest 字段差异 | BFF 层对齐现有 `{code,message,data}` 与 DTO |
| 4 | 迁移期 monolith 与微服务双维护 | 迁完一域即从 monolith 删代码，缩短窗口 |
