# Plan 12：定时任务与运维后台

## 元信息

| 项 | 内容 |
|----|------|
| **目标** | 迁移 Nest `scheduled-task` 模块：admin API + 8 个内置业务 cron + 执行日志 + DB 备份管理 |
| **前置依赖** | [11-微服务代码物理拆分.md](./11-微服务代码物理拆分.md) 验收通过 |
| **周期** | ~2–2.5 周 |
| **架构形态** | 4 微服务（**blog-service** 承载；gateway 代理 `/scheduled-task/*`） |
| **里程碑** | M6 Nest 差异补齐 |
| **Nest 对照** | `blog-server/src/modules/features/scheduled-task/` |

## 背景

Plan 04 仅接入 `robfig/cron/v3` 占位 job；Plan 09 补充 RPG 活动通知 08:00。Nest 另有 **8 个种子任务**、完整 admin CRUD/启停/手动触发、执行日志与备份下载，Go 侧 Ent 已有 `scheduled_task` / `scheduled_task_log` 表但 **无 HTTP 与业务 job**。

| GET | `/status/:taskName` | 运行状态 |

## 8 个内置种子任务（对齐 Nest `SEED_TASKS`）

| name | cron（6 段含秒） | 说明 |
|------|------------------|------|
| `daily_interaction_notify` | `0 0 10 * * *` | 每日互动邮件汇总 |
| `scheduled_publish` | `0 * * * * *` | 每分钟扫描 `scheduled` 文章发布 |
| `monthly_report` | `0 0 10 1 * *` | 每月 1 号博客月报邮件 |
| `link_health_check` | `0 0 3 * * 1` | 每周一友链健康检测 + 邮件 |
| `expired_data_cleanup` | `0 30 2 * * *` | 过期 operation_log 等清理 |
| `database_backup` | `0 0 4 * * *` | mysqldump 到 `backups/` |
| `draft_cleanup` | `0 0 5 * * *` | 90 天草稿清理 |
| `sensitive_word_alert` | `0 0 9 * * *` | 敏感词 pending 告警邮件 |

启动时 **幂等 upsert** 种子（Nest `onModuleInit` 行为）；禁止重复插入同名任务。

## 关键实现要点

### Scheduler 重构

- 替换 `RegisterPlaceholder`：按 DB `scheduled_task.enabled` 动态注册/注销 cron
- 每次 trigger/stop/start 写 `scheduled_task_log`（`logRecording=true` 时）
- 任务执行 panic recover + 错误写入 log.status

### `scheduled_publish`

- 查询 `status=scheduled` 且 `publishTime <= now` 的文章 → 改 `publish`、发 `blog.article.published` 事件

### `database_backup`

- 备份目录默认 `{cwd}/backups`，文件名 `myblog_backup_{timestamp}.sql`
- 校验下载文件名白名单（防路径穿越，对齐 Nest `backupNamePattern`）

### 权限

- 普通任务 API：RBAC 权限码与 Nest/admin 菜单一致（需在 `x_privilege` patch 或 fallback 登记）
- 备份下载 / clear-permissions / refresh-tongji-token：**仅超级管理员**（对照 Nest `SuperAdminGuard`）

## 风险与注意点

| 风险 | 对策 |
|------|------|
| Windows 无 `mysqldump` | 配置 `backup.mysqldump_path`；CI 用 Linux；文档说明 |
| 邮件 job 依赖 SMTP | 与 Plan 02 email 配置共用；未配置时 job 记 failed 不发 panic |
| 多实例重复 cron | 单实例 PM2 或 Redis 分布式锁（MVP 先单实例，文档标注） |
| blog/user 跨服务邮件 | 优先 gRPC 扩展，避免 blog 直连 SMTP 双份配置 |
