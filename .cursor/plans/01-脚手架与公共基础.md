## 关键实现要点

### 统一响应（保持前端兼容）

```go
// pkg/response/response.go
type Body struct {
    Code    int         `json:"code"`
    Message string      `json:"message"`
    Data    interface{} `json:"data,omitempty"`
}
```

HTTP 状态码始终 200，业务错误通过 `code` 字段表达（与 NestJS 一致）。

### 密码兼容

- 旧数据：crypto-js / node-jsencrypt 哈希 — 登录时验证通过后静默升级为 bcrypt
- 新注册：直接 bcrypt

### wire 装配

- `services/monolith/internal/app/wire.go` + `wire_gen.go`
- 注入：Config、Logger、Ent Client、Redis Client、Hertz Server

## NestJS 对照

本阶段不涉及业务模块迁移，仅需对照现网 MySQL 表结构（TypeORM entities）生成 Ent schema。

## 验收标准

- [ ] `go run services/monolith/cmd/main.go` 启动无 panic
- [ ] `curl localhost:5000/health` 或 `/api/v1/health` 返回 200 + `code:0`
- [ ] Ent client 查询任意一张表成功（如 `user`）
- [ ] Redis ping 成功
- [ ] `make wire` / `make ent-gen` 可重复执行

### 可脚本化验收

```bash
make dev   # 或 go run services/monolith/cmd/main.go
curl -sf http://localhost:5000/health | grep -q '"code":0'
curl -sf http://localhost:5000/api/v1/health | grep -q '"code":0'
make wire && make ent-gen
```

## 风险与注意点

| 风险 | 对策 |
|------|------|
| entimport 字段映射错误 | 逐表对照 TypeORM entity 人工校对 |
| 错误码遗漏 | 从 NestJS `common/constants/error-code` 批量迁移 |
| 过早拆微服务 | 本阶段严格单体，proto 仅占位 |
