## 模块范围

跨 user / blog 域的运维支撑模块；WS 推送逻辑在 Plan 08 实现。

### NestJS 对照

| Go 包 | NestJS 路径 |
|-------|-------------|
| `internal/user/sensitive` | `blog-server/src/modules/features/sensitive-word/` |
| `internal/blog/notification` | `blog-server/src/modules/features/notification/` |
| `internal/blog/operationlog` | `blog-server/src/modules/features/operation-log/` |
| `internal/blog/scheduler` | `blog-server/src/modules/features/scheduled-task/` |

### 负责表

- user 域：`blog_sensitiveword`, `blog_sensitivewordhit`（Ent schema 命名以现网为准）
- blog 域：`notification`, `operation_log`

## API 路径（保持兼容）

| 路径 | 说明 |
|------|------|
| `/api/v1/admin/sensitive-word/*` | 敏感词 CRUD（路径以 NestJS 为准） |
| `/api/v1/notification/*` | 通知 CRUD（推送在 Plan 08） |

## 关键实现要点

### 敏感词过滤

- CRUD + 过滤工具函数，供 comment/msgboard 等模块调用
- 命中记录写入 sensitivewordhit 表

### notification

- CRUD + 分页列表
- `GET /api/v1/notification/since?seq=` 接口骨架（Plan 08 完善 WS 补漏）

### operation-log

- admin 操作拦截器/中间件记录
- 列表查询接口

### 定时任务（骨架）

- `robfig/cron/v3` 接入
- 注册占位 job；RPG/通知相关 cron 在 Plan 09 完善
