## 目标架构

```
Nginx → api-gateway :8000 (REST + JWT + 限流 + BFF)
           ├── gRPC → blog-service  :5001 (含 WS Hub)
           ├── gRPC → user-service  :5002
           └── gRPC → rpg-service   :5003
                    ↓
              MySQL 8 + Redis 7（共享单库）
```

### gRPC 服务定义（摘录）

```protobuf
// proto/user/v1/user.proto
service UserService {
  rpc GetUser(GetUserReq) returns (User);
  rpc GetUserBatch(GetUserBatchReq) returns (UserBatch);
  rpc VerifyToken(VerifyTokenReq) returns (VerifyTokenResp);
}
```

### gateway BFF 聚合

```go
// services/gateway/internal/aggregator/article.go
// 1. blog-service.GetArticle
// 2. user-service.GetUser(authorID)
// 3. 合并返回
```

### 认证透传

- gateway：JWT 本地验签 → gRPC metadata `x-user-id`
- 内部服务：AuthInterceptor 从 metadata 取 userID，不重复验签

### API 路径 → 服务映射（附录 A）

| 原路径 | 服务 |

| `/api/v1/auth` `/user` `/captcha` `/admin/*` | user-service |

| `/api/v1/pay` `/rpg` `/admin/rpg` | rpg-service |
| `/api/v1/pub` | gateway 聚合 |

### 2G 服务器部署（总方案第七章）

**MySQL 调优**（`deploy/docker/mysql.cnf`）：

```ini
innodb_buffer_pool_size = 128M
max_connections = 100
performance_schema = OFF
```

**Go 连接池**（每服务）：

```go
db.SetMaxOpenConns(15)  // 4×15=60 < MySQL 100
db.SetMaxIdleConns(5)
```

**Docker 内存限制**：

```dockerfile
ENV GOMEMLIMIT=80MiB
ENV GOGC=50
```

**docker-compose**：gateway + blog + user + rpg + mysql + redis（总方案 7.3）

**Dockerfile**：multi-stage + distroless，单服务镜像 < 25MB
