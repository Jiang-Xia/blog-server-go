| 项 | 内容 |

| **前置依赖** | [11-微服务代码物理拆分.md](./11-微服务代码物理拆分.md) 验收通过；与 Plan 13 无硬依赖，可并行 |
| **周期** | ~3–5 天 |

| **里程碑** | M6 Nest 差异补齐 |
| **Nest 对照** | `modules/rpg/profile/profile.service.ts` → `getPublicCollectArticles` / `getPublicLikeArticles` |

## 背景

`services/rpg/internal/app/adapters.go` 中：

```go
GetPublicCollectArticles → 固定空 list
GetPublicLikeArticles    → 固定空 list
```

`GetPublicArticles` 已通过 `ListPublishedByAuthor` 实现。home/uniapp 公开主页 Tab 依赖收藏/点赞列表。

## 模块范围

### 数据规则（对齐 Nest）

**收藏** `collect` JOIN `article` JOIN `user(author)`：

- `col.uid = :uid`
- 文章 `isDelete=false`、`status=publish`
- 作者 `status=ACTIVE`、`isDelete=false`
- 按 `col.createTime DESC` 分页

**点赞** `like` JOIN `article` JOIN `author`：

- `like.uid = :uid`、`like.status=1`
- 同上文章/作者过滤
- 按 `like.id DESC` 分页

**响应字段**：与 Nest `mapPublicArticle` 一致（id、title、description、cover、views、likes、createTime 等；禁止前端 dict 回显额外字段）。

### 跨服务设计

collect/like/article 表归属 **blog-service**（Ent 边界）。rpg-service **禁止**直连 blog 表。

推荐：

1. 在 `proto/blog/v1/` 增加 RPC：
   - `ListPublicCollectArticles(uid, page, pageSize)`
   - `ListPublicLikeArticles(uid, page, pageSize)`
2. rpg-service `profile` 包调用 gRPC，删除 `adapters.go` stub
3. gateway 现有 `GET /user/public/:uid` BFF 不变；collects/likes 仍代理 rpg-service

备选（不推荐）：gateway BFF 聚合 blog gRPC — 仅当 rpg 路由迁移成本高时采用。

## API 路径

| 方法 | 路径 | 服务 |
|------|------|------|
| GET | `/api/v1/user/public/:uid/collects` | rpg-service（内部调 blog gRPC） |
| GET | `/api/v1/user/public/:uid/likes` | 同上 |

Query：`page`、`pageSize`（默认 1 / 10）。
